## Before Asking for Help

<Steps>
  <Step title="Check the FAQ">
    Most common questions are answered in the [FAQ](/faq). Check there first.
  </Step>

  <Step title="Review the Glossary">
    Unfamiliar terms? See the [Glossary](/glossary) for simple explanations.
  </Step>

  <Step title="Check Logs">
    Include relevant error messages from Docker logs when asking for help:

    ```bash
    docker compose logs bitbonsai-backend
    docker compose logs bitbonsai-frontend
    ```
  </Step>

  <Step title="Share Your Setup">
    When asking for help, include:
    - Operating system (Unraid, Ubuntu, Windows WSL2, etc.)
    - Docker version (`docker --version`)
    - BitBonsai version (see Settings → About in UI)
    - Relevant docker-compose.yml configuration
  </Step>
</Steps>

## Self-Service Resources

<AccordionGroup>
  <Accordion title="Installation Problems" icon="download">
    **Common issues:**
    - Container won't start → Check [Docker Installation Troubleshooting](/installation/docker#troubleshooting)
    - Permission errors → See [Volume Mount Setup](/installation/docker#volume-mounts)
    - GPU not detected → Review [Hardware Acceleration Guide](/installation/requirements#hardware-transcoding-optional)

    **Where to get help:**
    - Discord #installation-help channel
    - GitHub Discussions → Installation category
  </Accordion>

  <Accordion title="Encoding Issues" icon="film">
    **Common issues:**
    - Jobs stuck in QUEUED → Check [Understanding Jobs](/guides/understanding-jobs)
    - Slow encoding speed → See [Performance Optimization](/advanced/troubleshooting)
    - Corrupted output files → Review [Self-Healing Systems](/guides/understanding-jobs#self-healing-recovery)

    **Where to get help:**
    - Discord #encoding-help channel
    - GitHub Discussions → Encoding category
  </Accordion>

  <Accordion title="Multi-Node Setup" icon="server">
    **Common issues:**
    - Child node won't connect → Check [Multi-Node Setup Guide](/advanced/multi-node)
    - NFS mount errors → See [Shared Storage Configuration](/advanced/multi-node#shared-storage-setup)
    - Load balancing not working → Review [Distribution System](/advanced/multi-node#job-distribution)

    **Where to get help:**
    - Discord #multi-node channel
    - GitHub Discussions → Multi-Node category
  </Accordion>

  <Accordion title="GPU Problems" icon="microchip">
    **Common issues:**
    - GPU not detected by container → See [Docker GPU Passthrough](/installation/requirements#docker-gpu-passthrough)
    - FFmpeg not using GPU → Check [Hardware Encoding Verification](/installation/requirements#hardware-transcoding-optional)
    - Poor GPU encoding quality → Review [Codec Selection Guide](/advanced/codec-selection)

    **Where to get help:**
    - Discord #gpu-help channel
    - GitHub Discussions → Hardware category
  </Accordion>
</AccordionGroup>

## Response Times

<Note>
**Expected response times:**

- **Discord**: Usually within hours during business hours (EST timezone)
- **GitHub Issues**: 1-3 business days
- **GitHub Discussions**: 1-2 business days

BitBonsai is maintained by a small team. For urgent production issues, use Discord for fastest response.
</Note>

## Commercial Support

BitBonsai is open source and free to use. For organizations requiring dedicated support, contact us via:

- **Email**: support@bitbonsai.io
- **GitHub Sponsors**: [Support the project](https://github.com/sponsors/bitbonsai)

Commercial support includes:
- Priority issue resolution
- Custom feature development
- Multi-node cluster setup assistance
- Performance tuning for large libraries (10k+ videos)
