# 📖 MediaInsight Code Conventions

This folder contains the development guidelines and coding standards for the MediaInsight project.

---

## 📚 Available Guidelines

### Frontend Standards
- **[Angular Guidelines](./angular-guidelines.md)** - Angular 19 with NgRx patterns
  - Effect → Service → Client → BO architecture
  - Standalone components with OnPush detection
  - Modern template syntax (@if, @for, @switch)
  - BEM SCSS methodology
  - Business Objects pattern

### Backend Standards
- **[NestJS Guidelines](./nestjs-guidelines.md)** - NestJS backend patterns
  - Module-based architecture
  - Controller → Service → Repository pattern
  - DTO validation and transformation
  - API documentation with Swagger
  - Error handling standards

### Testing Standards
- **[Testing Guidelines](./testing-guidelines.md)** - Testing requirements and patterns
  - Unit testing strategies
  - Integration testing approaches
  - Test coverage standards
  - Mocking best practices

### Internationalization
- **[i18n Guidelines](./i18n-guidelines.md)** - Translation and localization standards
  - Translation key naming conventions
  - File organization
  - Pluralization patterns

### Version Control
- **[Git Commit Instructions](./git-commit-instructions.md)** - Git workflow and commit standards
  - Commit message format
  - Branch naming conventions
  - PR guidelines
