LXC Container Detected

Your child node is running in an LXC container without privileged mode. You have three options:

  • Option 1: Enable privileged mode on the LXC container
  • Option 2: Mount NFS on the Proxmox host and bind-mount into container
  • Option 3: Use rsync file transfer (automatic, no setup needed)

See LXC Configuration Guide for detailed instructions.

NFS Configuration

Step 1: Enable NFS on Main Node

On Unraid (Main Node):

  1. Go to Settings → NFS
  2. Enable NFS Server: Yes
  3. Add media shares to exports
  4. Click Apply

Step 2: Verify NFS Exports

showmount -e 192.168.1.100

Expected output:
/mnt/user/media *
/mnt/user/Downloads *

Step 3: Enable LXC Privileged Mode

# Stop container
pct stop <container-id>

# Edit config
nano /etc/pve/lxc/<container-id>.conf

# Add this line:
unprivileged: 0

# Save and start
pct start <container-id>
Security Note: Privileged mode gives the container root access. Only use for trusted encoding nodes on isolated networks.

rsync File Transfer

Good news! rsync file transfer requires no configuration.

How It Works

  1. Job Assigned: When a job is assigned to {{ targetNodeName }}
  2. File Transfer: Source file is transferred via rsync (~1-2 min for 10GB)
  3. Encoding: FFmpeg processes the file locally on {{ targetNodeName }}
  4. Result Transfer: Encoded file is transferred back to {{ sourceNodeName }}
  5. Cleanup: Temp files are automatically removed
